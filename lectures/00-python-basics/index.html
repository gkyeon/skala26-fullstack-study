💡
쉽게 말하면: Python은 변수에 =로 값을 바로 담을 수 있고, 자료형을 미리 선언하지 않아도 돼요. 리스트는 내용물을 바꿀 수 있는 상자, 튜플은 한 번 담으면 못 바꾸는 봉인된 상자, 딕셔너리는 이름표가 붙은 서랍장, 집합은 중복을 자동으로 걸러주는 통이라고 생각하면 돼요.

변수와 기본 데이터 타입

  • 변수는 등호(=)로 값을 할당하며, 자료형을 미리 선언할 필요가 없다(동적 타이핑)
  • 정수(int)는 크기 제한이 없고, 실수(float)는 소수점 이하 값을 포함하며, 정수·실수 연산 시 결과는 자동으로 실수로 형 변환된다
  • 복소수(complex, 2+3j), 불리언(bool, True/False)도 기본 자료형에 포함
  • 사칙연산 + - * /, 몫 연산자 //, 나머지 연산자 %, 제곱 연산자 **
  • type()으로 자료형 확인, import keyword; keyword.kwlist로 예약어 목록 확인 가능

문자열 — 인덱싱·슬라이싱·포맷팅

  • 문자열은 큰따옴표·작은따옴표 모두 사용 가능, 따옴표 자체를 출력하려면 역슬래시(\)로 이스케이프
  • 슬라이싱 문법: 문자열[start:end:step] — end는 포함되지 않음, step에 -1을 주면 뒤집기
  • 문자열끼리 + 연산 시 단순 연결, 문자열은 불변(Immutable) 객체
  • 포맷팅 방식은 4가지: % 연산자 → .format() → f-string 순으로 발전, f-string이 가장 현대적

  • count()·index() 메서드, len/max/min/sum/sorted() 내장 함수와도 함께 사용

딕셔너리(Dictionary) — Key-Value 저장소

  • {key: value} 형식, 키는 고유해야 하지만 값은 중복 가능
  • Python 3.7부터 삽입 순서를 보장한다는 점이 중요
  • 접근은 d[key](없으면 KeyError) 또는 d.get(key, 기본값)(없으면 None/기본값)
  • 삭제는 del d[key]·pop(key)·popitem()·clear()
  • 딕셔너리 컴프리헨션: {키_표현식: 값_표현식 for 변수 in 반복가능 if 조건} — 변환·필터링에 매우 유용
딕셔너리 — 생성·CRUD·컴프리헨션

person.update({"age": 30, "job": "Engineer"})

집합(Set)과 frozenset

  • 중복을 허용하지 않고 순서가 없는 자료형, {}는 빈 딕셔너리이므로 빈 집합은 set()로 생성
  • 집합 연산: 합집합 |, 교집합 &, 차집합 -, 대칭차집합 ^
  • frozenset은 불변 집합 — 딕셔너리의 키나 집합의 원소로 사용 가능(일반 set은 해시 불가라 불가능)
  • Bool 자료형: 0, 빈 리스트/튜플/딕셔너리, None 등은 False, 나머지는 True
집합 연산 & frozenset
set_a = {1, 2, 3}
set_b = {3, 4, 5}
print(set_a | set_b)  # 합집합: {1,2,3,4,5}
print(set_a & set_b)  # 교집합: {3}
print(set_a - set_b)  # 차집합: {1,2}

fs1 = frozenset([1, 2, 3, 4])
fs2 = frozenset([3, 4, 5, 6])
print(fs1 ^ fs2)  # 대칭차집합: frozenset({1,2,5,6})

# frozenset은 딕셔너리 키로 사용 가능 (일반 set은 불가)
discounts = {frozenset(["VIP", "Holiday"]): 20}
print(discounts[frozenset(["Holiday", "VIP"])])  # 20 (순서 무관)

2. 함수와 모듈

함수 정의·호출 · 기본값·가변 인자 · 람다·고차함수 · 모듈과 표준 라이브러리

💡
쉽게 말하면: 함수는 "입력을 넣으면 정해진 일을 하고 결과를 돌려주는 상자"예요. *args는 인자를 몇 개 받을지 모를 때 튜플로 몽땅 받는 것, **kwargs는 이름표 붙은 인자를 딕셔너리로 몽땅 받는 것이에요. 람다는 이름 없이 한 줄로 쓰는 미니 함수고요.

함수 정의·호출과 매개변수 기본값

  • def 함수이름(매개변수): ... return 결과 형태로 정의, return이 없으면 None 반환
  • 매개변수에 기본값을 줄 수 있다(def introduce(name, age=20)) — 단, 기본값 있는 인자는 항상 뒤쪽에 배치해야 한다(안 그러면 SyntaxError)
함수 정의 & 기본값 매개변수
def greet(name):
    return f"안녕하세요, {name}님!"
print(greet("철수"))  # 안녕하세요, 철수님!

def introduce(name, age=20):        # 기본값은 뒤쪽 인자에만
    print(f"이름: {name}, 나이: {age}")
introduce("영희")        # 나이: 20 (기본값 사용)
introduce("철수", 25)    # 나이: 25

가변 인자 — *args, **kwargs

  • *args: 여러 개의 위치 인자를 튜플로 저장, **kwargs: 여러 개의 키워드 인자를 딕셔너리로 저장
  • 함께 사용할 때 순서는 반드시 일반 인자 → *args → **kwargs(반대로 쓰면 SyntaxError)
*args / **kwargs 혼합 사용
def mix_example(a, b, *args, **kwargs):
    print(f"a:{a}, b:{b}")
    print(f"args:{args}")      # 튜플
    print(f"kwargs:{kwargs}")  # 딕셔너리

mix_example(1, 2, 3, 4, 5, name="철수", age=30)
# a:1, b:2
# args:(3, 4, 5)
# kwargs:{'name': '철수', 'age': 30}

def sum_numbers(*args):
    return sum(args)
print(sum_numbers(1, 2, 3, 4, 5))  # 15

5. 객체지향 프로그래밍 기초

클래스와 객체 · self · 생성자 · 상속 · 메서드 오버라이딩 · super()

💡
쉽게 말하면: 클래스는 붕어빵 틀, 인스턴스(객체)는 그 틀에서 찍어낸 붕어빵이에요. self는 "지금 만들어지고 있는 이 붕어빵 자신"을 가리키는 말이고, 상속은 이미 있는 틀을 베이스로 새로운 틀을 만드는 것, 오버라이딩은 부모 틀의 기능을 자식 틀에서 다시 정의하는 거예요.

클래스와 객체 — self로 속성 접근

  • class 클래스이름:으로 정의, __init__(self, ...)는 객체 생성 시 자동 호출되는 생성자
  • self는 "현재 만들어진 객체 자신"을 가리키며, 인스턴스 속성(self.속성 = 값)과 메서드를 정의할 때 첫 번째 인자로 항상 필요
  • 같은 클래스에서 만든 객체(인스턴스)들은 서로 독립적인 속성 값을 가진다

  • 메서드 안에서 조건 분기로 잔액 검증 같은 도메인 규칙을 캡슐화할 수 있다

        self.owner = owner
        self.balance = balance

상속과 메서드 오버라이딩

  • 상속: class Dog(Animal):처럼 부모 클래스(Super Class)의 속성·메서드를 자식 클래스(Sub Class)가 그대로 물려받는다
  • 오버라이딩: 자식 클래스에서 부모와 같은 이름의 메서드를 다시 정의해 동작을 덮어씀
  • super().메서드()를 호출하면 부모의 동작을 유지하면서 자식에서 기능을 추가할 수 있다
상속 · 오버라이딩 · super()
class Animal:
    def __init__(self, name):
        self.name = name
    def speak(self):
        return "소리를 낸다."

class Dog(Animal):          # Animal을 상속
    def speak(self):        # 오버라이딩
        sound = super().speak()          # 부모 메서드 호출
        return sound + " 하지만 나는 멍멍 짖는다!"

class Cat(Animal):
    def speak(self):
        return "야옹!"

dog = Dog("바둑이")
print(dog.name)     # 바둑이 (부모 속성 사용 가능)
print(dog.speak())  # 소리를 낸다. 하지만 나는 멍멍 짖는다!
print(Cat("나비").speak())  # 야옹!
오버라이딩 시 super()를 호출하지 않으면 부모의 동작은 완전히 사라지고 자식의 새 동작으로 대체된다. "부모 기능을 유지하며 확장"하려면 반드시 super()를 사용해야 한다.

6. 고급 자료구조

collections(deque·Counter·defaultdict·OrderedDict) · heapq · bisect

💡
쉽게 말하면: 리스트는 끝에서 추가/삭제(O(1))는 빠르지만 앞에서 하면 느려요(O(n)). deque는 양쪽 끝 모두 빠른(O(1)) 큐예요. heapq는 "가장 작은 값을 항상 맨 앞에 유지"해주는 우선순위 큐고, bisect는 정렬된 리스트에서 값이 들어갈 위치를 빠르게 찾아줘요.

왜 collections 모듈을 쓰는가 — 리스트의 한계

  • 리스트는 동적 배열(Dynamic Array) 구조라 끝에서 추가/삭제는 O(1)이지만, 맨 앞에서 삽입/삭제(insert(0, ...), pop(0))는 나머지 요소를 모두 밀어야 해서 O(n)
  • deque(collections.deque)는 이중 연결 리스트 구조라 양쪽 끝 모두 O(1) — 슬라이딩 윈도우, 큐/스택 구현에 적합
deque — 양방향 O(1) 큐
from collections import deque
dq = deque([1, 2, 3])
dq.append(4)       # 오른쪽 추가, O(1)
dq.appendleft(0)    # 왼쪽 추가, O(1)
print(dq)  # deque([0, 1, 2, 3, 4])
dq.pop()            # 오른쪽 제거
dq.popleft()        # 왼쪽 제거
print(dq)  # deque([1, 2, 3])

Counter, defaultdict, OrderedDict

  • Counter: 빈도 집계 전문 — most_common(n)으로 상위 n개 확인
  • defaultdict: 키가 없을 때 if key not in dict 없이 기본값을 자동 생성
  • OrderedDict: 삽입 순서를 보장하는 dict(Python 3.7+ 기본 dict와 동일하게 동작하지만 의도를 명시할 때 사용)
Counter · defaultdict

print(count)  # {'l':3, 'o':2, 'h':1, ...}

word_count = defaultdict(int)  # 기본값 0
word_count["apple"] += 1
word_count["banana"] += 2
print(word_count)  # {'apple': 1, 'banana': 2}

heapq — 우선순위 큐(최소 힙)

  • heapq는 항상 가장 작은 값을 O(log n)에 꺼낼 수 있는 최소 힙 구현 — 다익스트라 최단 경로, 작업 스케줄링에 필수
  • 최대 힙이 필요하면 값을 음수로 뒤집어 넣고 꺼낼 때 다시 부호를 반전
heapq — 최소 힙 / 최대 힙
import heapq
heap = []
for v in [5, 2, 8, 1]:
    heapq.heappush(heap, v)
print(heapq.heappop(heap))  # 1 (가장 작은 값)

max_heap = []
for v in [5, 1, 10]:
    heapq.heappush(max_heap, -v)  # 부호 반전
print(-heapq.heappop(max_heap))  # 10 (가장 큰 값)

# 다익스트라 알고리즘: (거리, 노드) 튜플을 힙에 넣어 최단거리 노드를 먼저 처리
def dijkstra(graph, start):
    distances = {node: float('inf') for node in graph}
    distances[start] = 0
    pq = [(0, start)]
    while pq:
        d, node = heapq.heappop(pq)
        if d > distances[node]:
            continue
        for nb, w in graph[node].items():
            nd = d + w
            if nd < distances[nb]:
                distances[nb] = nd
                heapq.heappush(pq, (nd, nb))
    return distances

bisect — 정렬된 리스트의 빠른 위치 탐색

  • bisect_left(list, x)는 이진 탐색으로 x가 들어갈 위치를 O(log n)에 찾는다
  • bisect.insort(list, x)는 정렬 순서를 유지하면서 값을 자동으로 삽입
bisect 이진 탐색 삽입
import bisect
numbers = [1, 3, 5, 7, 9]
pos = bisect.bisect_left(numbers, 6)
print(pos)  # 3 (6이 들어갈 인덱스)
bisect.insort(numbers, 6)
print(numbers)  # [1, 3, 5, 6, 7, 9]

7. 데코레이터와 클로저

데코레이터 · 클로저 · 데코레이터 체이닝 · functools.wraps

💡
쉽게 말하면: 데코레이터는 함수에 옷을 입히듯 기능을 덧씌우는 거예요(실행시간 측정, 로그인 체크 등). 클로저는 내부 함수가 자신을 만든 바깥 함수의 변수를 "기억"하는 성질이에요. 데코레이터는 사실 클로저를 활용해서 만들어져요.

기본 데코레이터 — 함수를 감싸는 함수

  • 데코레이터는 함수를 인자로 받아 새로운 함수(wrapper)를 반환하는 함수 — @데코레이터이름을 함수 위에 붙여 적용
  • 내부 wrapper 함수는 *args, **kwargs로 원래 함수의 인자를 그대로 전달받아야 범용적으로 쓸 수 있다
기본 데코레이터 패턴
def my_decorator(func):
    def wrapper(*args, **kwargs):
        print("함수 실행 전")
        result = func(*args, **kwargs)
        print("함수 실행 후")
        return result
    return wrapper

@my_decorator
def add(a, b):
    return a + b

print(add(3, 5))
# 함수 실행 전
# 함수 실행 후
# 8

실용적인 데코레이터 — @timer, @require_login

  • 실행 시간 측정, 로그인 여부 확인처럼 여러 함수에 공통으로 필요한 로직을 데코레이터로 분리하면 중복을 없앨 수 있다
@timer / @require_login
import time
def timer(func):
    def wrapper(*args, **kwargs):
        start = time.time()
        result = func(*args, **kwargs)
        print(f"실행 시간: {time.time()-start:.4f}초")
        return result
    return wrapper

@timer
def slow_function():
    time.sleep(2)
    print("작업 완료!")

def require_login(func):
    def wrapper(user):
        if user.get("is_logged_in"):
            return func(user)
        print("로그인이 필요합니다.")
    return wrapper

@require_login
def dashboard(user):
    print(f"{user['name']}님의 대시보드입니다.")

클로저(Closure) — 외부 변수를 기억하는 내부 함수

  • 클로저는 내부 함수가 자신을 감싸는 외부 함수의 지역 변수를 참조·기억하는 특성 — 데코레이터의 wrapper가 바로 클로저다
클로저 예제
def make_multiplier(n):
    def multiply(x):
        return x * n     # 바깥 함수의 n을 계속 기억
    return multiply

double = make_multiplier(2)
triple = make_multiplier(3)
print(double(5))  # 10
print(triple(5))  # 15 — 각자 다른 n을 독립적으로 기억

데코레이터 체이닝과 functools.wraps

  • 여러 데코레이터를 동시에 적용할 수 있으며, 실행 순서는 가장 가까운 데코레이터부터 안쪽으로(아래에서 위로 감싸짐)
  • 데코레이터를 쓰면 원본 함수의 __name__·__doc__이 wrapper로 덮여 사라지는 문제가 생긴다 — @functools.wraps(func)를 wrapper 위에 붙이면 원본 메타데이터가 유지된다
데코레이터 체이닝 + functools.wraps
import functools

def decorator1(func):
    @functools.wraps(func)
    def wrapper(*args, **kwargs):
        print("[decorator1] 실행 전")
        result = func(*args, **kwargs)
        print("[decorator1] 실행 후")
        return result
    return wrapper

def decorator2(func):
    @functools.wraps(func)
    def wrapper(*args, **kwargs):
        print("[decorator2] 실행 전")
        result = func(*args, **kwargs)
        print("[decorator2] 실행 후")
        return result
    return wrapper

@decorator1
@decorator2
def say_hello():
    """이 함수는 인사를 합니다."""
    print("안녕하세요!")

say_hello()
# [decorator1] 실행 전 → [decorator2] 실행 전 → 안녕하세요!
# → [decorator2] 실행 후 → [decorator1] 실행 후
print(say_hello.__name__)  # say_hello (wraps 덕분에 이름 유지)

8. 멀티스레딩과 멀티프로세싱

threading · multiprocessing · GIL(Global Interpreter Lock)

💡
쉽게 말하면: 멀티스레딩은 한 프로그램 안에서 여러 작업을 동시에 진행하는 것(가벼움), 멀티프로세싱은 아예 별도의 프로그램(프로세스)을 여러 개 띄우는 것(무겁지만 진짜 병렬)이에요. Python은 GIL이라는 잠금장치 때문에 CPU 계산이 많은 작업은 스레드로는 진짜 병렬 처리가 안 돼요 — 이럴 땐 멀티프로세싱을 써야 해요.

멀티스레딩(Multithreading)

  • threading.Thread(target=함수)로 스레드를 생성, .start()로 실행, .join()으로 종료를 기다림
  • 스레드는 메모리를 공유하기 때문에 가볍지만, I/O 대기(파일·네트워크)가 많은 작업에 효과적
threading — 두 작업을 동시에
import threading, time

def print_numbers():
    for i in range(1, 6):
        print(i); time.sleep(1)

def print_letters():
    for c in "ABCDE":
        print(c); time.sleep(1)

t1 = threading.Thread(target=print_numbers)
t2 = threading.Thread(target=print_letters)
t1.start(); t2.start()
t1.join(); t2.join()   # 두 스레드가 끝날 때까지 대기

멀티프로세싱(Multiprocessing)

  • multiprocessing.Process(target=함수)로 완전히 독립된 프로세스를 생성 — 메모리를 공유하지 않아 진짜 병렬 실행이 가능
  • CPU를 많이 쓰는 연산(이미지 처리, 대량 계산)에는 멀티프로세싱이 유리하다
multiprocessing — 완전 독립 병렬 실행
import multiprocessing, time

def print_numbers():
    for i in range(1, 6):
        print(i); time.sleep(1)

def print_letters():
    for c in "ABCDE":
        print(c); time.sleep(1)

p1 = multiprocessing.Process(target=print_numbers)
p2 = multiprocessing.Process(target=print_letters)
p1.start(); p2.start()
p1.join(); p2.join()

Global Interpreter Lock(GIL)

  • GIL은 한 번에 하나의 스레드만 Python 바이트코드를 실행하도록 제한하는 CPython의 잠금장치
  • 그래서 CPU-bound(계산 위주) 작업은 멀티스레딩으로 진짜 병렬화되지 않는다 → 멀티프로세싱 사용
  • I/O-bound(대기 위주, 네트워크·파일) 작업은 대기 중 GIL이 풀리므로 멀티스레딩으로도 충분히 효과적
  • NumPy 같은 라이브러리는 내부 C 코드 실행 중 GIL을 해제하기 때문에, NumPy 연산은 스레드를 써도 실제 병렬 이득을 볼 수 있다

9. 동적 프로그래밍 기법 (Dynamic Programming)

메모이제이션(Memoization) · 타뷸레이션(Tabulation) · 점화식

💡
쉽게 말하면: 동적 프로그래밍은 "이미 풀었던 작은 문제의 답은 다시 계산하지 않고 저장해뒀다가 재사용"하는 기법이에요. 피보나치를 재귀로만 풀면 같은 값을 수백만 번 다시 계산하는데, 한 번 계산한 값을 저장해두면(메모이제이션) 순식간에 끝나요.

왜 필요한가 — 순수 재귀의 비효율

  • 피보나치 수열 fib(n) = fib(n-1) + fib(n-2)를 그대로 재귀로 구현하면 같은 하위 문제를 지수적으로 반복 계산해 n이 커질수록 급격히 느려진다
순수 재귀 (비효율)
def fib(n):
    if n <= 1:
        return n
    return fib(n-1) + fib(n-2)
print(fib(10))  # 55 (n이 커지면 매우 느려짐)

메모이제이션(Memoization) — 위에서 아래로(Top-down)

  • 계산한 결과를 딕셔너리 등에 저장해두고, 다음에 같은 입력이 들어오면 재계산 없이 바로 반환
메모이제이션 — dict에 결과 캐싱
def fib_memo(n, memo={}):
    if n in memo:
        return memo[n]
    if n <= 1:
        return n
    memo[n] = fib_memo(n-1, memo) + fib_memo(n-2, memo)
    return memo[n]
print(fib_memo(50))  # 즉시 계산 완료

타뷸레이션(Tabulation) — 아래에서 위로(Bottom-up)

  • 작은 문제부터 순서대로 배열(dp)에 저장하며 반복문으로 쌓아 올리는 방식 — 재귀 호출 스택이 없어 더 안전하고 빠르다
  • 공간을 더 절약하려면 직전 두 값만 유지하는 방식(a, b = b, a+b)으로 최적화할 수 있다
타뷸레이션 & 공간 최적화
def fib_tab(n):
    dp = [0] * (n+1)
    dp[1] = 1
    for i in range(2, n+1):
        dp[i] = dp[i-1] + dp[i-2]   # 점화식
    return dp[n]

def fib_optimized(n):      # 공간 복잡도 O(1)
    a, b = 0, 1
    for _ in range(n):
        a, b = b, a + b
    return a

DP 응용 예제 — 계단 오르기, 동전 교환

  • "계단 오르기"는 dp[i] = dp[i-1] + dp[i-2] 형태로 피보나치와 동일한 점화식 구조
  • "최소 동전 개수"는 각 금액에 대해 사용 가능한 동전 단위마다 min()으로 최적값을 갱신
계단 오르기 & 최소 동전 개수
def count_ways(n):   # 계단 오르기 경우의 수
    dp = [0] * (n+1)
    dp[0], dp[1] = 1, 1
    for i in range(2, n+1):
        dp[i] = dp[i-1] + dp[i-2]
    return dp[n]

def min_coins(n):    # 1,3,4원 동전으로 n원 만들 최소 개수
    dp = [float('inf')] * (n+1)
    dp[0] = 0
    for i in range(1, n+1):
        if i >= 1: dp[i] = min(dp[i], dp[i-1] + 1)
        if i >= 3: dp[i] = min(dp[i], dp[i-3] + 1)
        if i >= 4: dp[i] = min(dp[i], dp[i-4] + 1)
    return dp[n]

10. 선형 자료구조 (Linear Data Structures)

스택 · 큐 · 링크드 리스트 · Python 리스트 vs NumPy 배열

💡
쉽게 말하면: 데이터가 한 줄로 이어진 구조가 선형 자료구조예요. 스택은 나중에 넣은 게 먼저 나가는 접시 쌓기(LIFO), 큐는 먼저 넣은 게 먼저 나가는 줄서기(FIFO), 링크드 리스트는 각 칸이 "다음 칸이 어디인지"만 알고 있는 기차 칸 연결이에요.

스택(Stack)과 큐(Queue)

  • 스택은 LIFO(Last In First Out) — 나중에 넣은 것이 먼저 나감, 파이썬 리스트의 append()/pop()으로 구현
  • 큐는 FIFO(First In First Out) — 먼저 넣은 것이 먼저 나감, collections.deque의 append()/popleft()로 구현(리스트의 pop(0)은 O(n)이라 비효율적)
스택(list) & 큐(deque)
# 스택 — LIFO
stack = []
stack.append(10); stack.append(20)   # push
print(stack.pop())  # 20 (나중에 넣은 게 먼저 나감)

# 큐 — FIFO
from collections import deque
queue = deque()
queue.append(10); queue.append(20)   # enqueue
print(queue.popleft())  # 10 (먼저 넣은 게 먼저 나감)

링크드 리스트(Linked List)

  • 각 노드(Node)가 데이터와 "다음 노드를 가리키는 포인터(next)"를 가지고 있는 구조 — 배열과 달리 메모리가 연속적이지 않아도 됨
  • 중간 삽입/삭제가 배열보다 유리하지만(포인터만 바꾸면 됨), 인덱스로 바로 접근할 수 없어 탐색은 O(n)
단일 링크드 리스트 구현
class Node:
    def __init__(self, data):
        self.data = data
        self.next = None

class LinkedList:
    def __init__(self):
        self.head = None
    def append(self, data):
        new_node = Node(data)
        if not self.head:
            self.head = new_node
            return
        temp = self.head
        while temp.next:
            temp = temp.next
        temp.next = new_node

ll = LinkedList()
ll.append(10)
ll.append(20)

Python 리스트 vs NumPy 배열 & 벡터화 연산

  • Python 리스트는 서로 다른 자료형을 섞어 담을 수 있지만, NumPy 배열(np.array)은 모두 같은 타입이라 수치 계산·대규모 데이터 처리에 훨씬 빠르고 효율적
  • 벡터화 연산: 반복문 없이 배열 전체를 한 번에 연산 — 내부적으로 C 레벨에서 처리되어 for 루프보다 훨씬 빠름
반복문 vs 벡터화 연산 비교
list1, list2 = [1, 2, 3], [4, 5, 6]
result = [list1[i] + list2[i] for i in range(len(list1))]  # 반복문
print(result)  # [5, 7, 9]

import numpy as np
arr1, arr2 = np.array([1, 2, 3]), np.array([4, 5, 6])
result = arr1 + arr2   # 벡터화 연산 — 훨씬 빠름
print(result)  # [5 7 9]

# 100만개 원소 제곱: 리스트 컴프리헨션 vs NumPy 벡터화 속도 비교
size = 1_000_000
py_list = list(range(size))

11. 비선형 자료구조 (Non-Linear Data Structures)

트리 · 이진 탐색 트리(BST) · 그래프 · 힙 · DFS/BFS

💡
쉽게 말하면: 비선형 자료구조는 데이터가 가지치기처럼 여러 방향으로 연결된 구조예요. 트리는 부모-자식 관계로 뻗어나가는 구조(가계도), 그래프는 노드끼리 자유롭게 연결된 네트워크(지하철 노선도)예요. DFS는 "일단 끝까지 파고들고" BFS는 "가까운 곳부터 층층이" 탐색해요.

트리(Tree)와 이진 탐색 트리(BST)

  • 트리는 노드가 부모-자식 관계로 계층적으로 연결된 구조 — 루트(root)에서 시작해 leaf까지 뻗어나감
  • 이진 탐색 트리(BST)는 왼쪽 자식은 항상 부모보다 작고, 오른쪽 자식은 항상 부모보다 큰 규칙을 가진 트리 — 탐색·삽입이 평균 O(log n)
이진 트리 & BST 구현
class Node:
    def __init__(self, value):
        self.value = value
        self.left = None
        self.right = None

root = Node(1)
root.left = Node(2)
root.right = Node(3)

class BST:
    def __init__(self, value):
        self.value = value
        self.left = None
        self.right = None
    def insert(self, value):
        if value < self.value:
            if self.left: self.left.insert(value)
            else: self.left = BST(value)
        else:
            if self.right: self.right.insert(value)
            else: self.right = BST(value)

root = BST(10)
root.insert(5)   # 왼쪽 (10보다 작음)
root.insert(15)  # 오른쪽 (10보다 큼)

그래프(Graph)와 힙(Heap)

  • 그래프는 파이썬에서 흔히 {노드: [인접노드, ...]} 형태의 딕셔너리로 표현
  • 힙은 항상 가장 작은(또는 큰) 값이 루트에 위치하는 완전 이진 트리 — Day 2에서 배운 heapq가 바로 이 구조를 구현한다
그래프(dict 표현) & heapq
graph = {1: [2, 3], 2: [4, 5], 3: [6], 4: [], 5: [], 6: []}
print(graph[1])  # [2, 3]

import heapq
heap = []
for v in [10, 5, 20]:
    heapq.heappush(heap, v)
print(heapq.heappop(heap))  # 5 (가장 작은 값)

그래프 탐색 — DFS vs BFS

  • DFS(깊이 우선 탐색): 한 방향으로 끝까지 파고든 뒤 되돌아오는 방식(재귀 또는 스택), 백트래킹 방식으로 동작하며 미로 탐색 같은 경로 문제에 적합
  • BFS(너비 우선 탐색): 가까운 노드부터 계층적으로 탐색(큐 사용), 최단 경로 문제에 유용
DFS(재귀) & BFS(큐)
def dfs(graph, node, visited):
    visited.add(node)
    print(node, end=" ")
    for neighbor in graph[node]:
        if neighbor not in visited:
            dfs(graph, neighbor, visited)

graph = {1: [2, 3], 2: [4, 5], 3: [6], 4: [], 5: [], 6: []}
dfs(graph, 1, set())  # 1 2 4 5 3 6

from collections import deque
def bfs(graph, start):
    queue = deque([start])
    visited = {start}
    while queue:
        node = queue.popleft()
        print(node, end=" ")
        for neighbor in graph[node]:
            if neighbor not in visited:
                queue.append(neighbor)
                visited.add(neighbor)

bfs(graph, 1)  # 1 2 3 4 5 6

12. 알고리즘 기초

개념과 종류 · 탐색·정렬 알고리즘 · 시간 복잡도 · 공간 복잡도

💡
쉽게 말하면: 시간 복잡도는 "입력이 커질수록 얼마나 오래 걸리는가"를 O(...) 표기법으로 나타낸 것이고, 공간 복잡도는 "얼마나 많은 메모리를 추가로 쓰는가"예요. O(1)은 항상 똑같이 빠르고, O(n)은 데이터 양에 비례해서 느려지고, O(n²)은 데이터가 늘면 급격히 느려져요.

시간 복잡도(Time Complexity) — Big-O 표기법

  • 알고리즘의 실행 속도를 입력 크기(n)에 대한 함수로 수학적으로 표현하는 방법, 보통 최악의 경우(worst-case)를 기준으로 분석
  • O(1) 상수 시간(입력 크기와 무관), O(log n) 로그 시간(이진 탐색), O(n) 선형 시간(전체 순회), O(n²) 이차 시간(이중 반복문)
O(1) vs O(n) vs O(n²) 예시
arr = [1, 2, 3, 4, 5]
print(arr[-1])  # O(1) — 인덱스 접근은 크기와 무관

def print_all(arr):        # O(n) — 선형 시간
    for x in arr:
        print(x)

def print_pairs(arr):      # O(n^2) — 이중 반복문
    for i in arr:
        for j in arr:
            print(i, j)

탐색 알고리즘 — 선형 탐색 vs 이진 탐색

  • 선형 탐색은 처음부터 끝까지 하나씩 비교 — O(n), 정렬 여부와 무관하게 사용 가능
  • 이진 탐색은 정렬된 리스트에서 중간값과 비교해 탐색 범위를 절반씩 줄여나감 — O(log n)으로 훨씬 빠름
선형 탐색 & 이진 탐색
def linear_search(arr, target):
    for i in range(len(arr)):
        if arr[i] == target:
            return i
    return -1

def binary_search(arr, target):   # arr는 반드시 정렬되어 있어야 함
    left, right = 0, len(arr) - 1
    while left <= right:
        mid = (left + right) // 2
        if arr[mid] == target:
            return mid
        elif arr[mid] < target:
            left = mid + 1
        else:
            right = mid - 1
    return -1

numbers = [10, 20, 30, 40, 50]
print(binary_search(numbers, 30))  # 2

정렬 알고리즘 — 버블 정렬 vs 퀵 정렬

  • 버블 정렬: 인접한 두 원소를 비교해 자리를 바꾸는 방식을 반복 — 구현은 쉽지만 O(n²)로 느림
  • 퀵 정렬: 기준값(pivot)보다 작은 값/큰 값으로 나눈 뒤 재귀적으로 정렬 — 평균 O(n log n)로 실무에서 널리 사용
버블 정렬 & 퀵 정렬
def bubble_sort(arr):
    n = len(arr)
    for i in range(n):
        for j in range(n - i - 1):
            if arr[j] > arr[j + 1]:
                arr[j], arr[j+1] = arr[j+1], arr[j]
    return arr

def quick_sort(arr):
    if len(arr) <= 1:
        return arr
    pivot = arr[len(arr) // 2]
    left = [x for x in arr if x < pivot]
    middle = [x for x in arr if x == pivot]
    right = [x for x in arr if x > pivot]
    return quick_sort(left) + middle + quick_sort(right)

numbers = [5, 3, 8, 4, 2]
print(bubble_sort(numbers.copy()))
print(quick_sort(numbers))

정렬 알고리즘 비교

알고리즘평균 시간복잡도공간복잡도특징
버블 정렬O(n²)O(1)구현이 단순하지만 느림, 교육용
퀵 정렬O(n log n)O(log n)~O(n)평균적으로 가장 빠름, pivot 선택에 민감
이진 탐색(정렬된 리스트)O(log n)O(1)정렬이 선행되어야 사용 가능

공간 복잡도(Space Complexity)

  • 알고리즘 실행에 추가로 필요한 메모리 양을 입력 크기(n)에 대한 함수로 표현
  • 기존 배열을 직접 수정하면(in-place) 공간 복잡도 O(1), 새로운 배열을 만들어 저장하면 O(n)
공간 복잡도 O(1) vs O(n) 비교
# 시간 O(n), 공간 O(1) — 추가 메모리 거의 없음
def print_elements(arr):
    for element in arr:
        print(element)

# 시간 O(n), 공간 O(n) — 새 리스트를 만들어 메모리 추가 사용
def duplicate_elements(arr):
    result = []
    for element in arr:
        result.append(element)
    return result

Pandas란? — Series와 DataFrame

  • Series: 인덱스가 붙은 1차원 배열(표의 한 열)
  • DataFrame: 여러 Series(컬럼)가 모인 2차원 테이블 — 딕셔너리로도 쉽게 생성 가능

[참고] 결측치(NaN) 처리

  • 제거는 dropna(), 채우기는 fillna(값) — 평균값·0·앞/뒤 값(ffill/bfill)으로 채울 수 있다

DataFrame 결합 — merge와 join

  • pd.merge(df1, df2, on='ID', how='inner')는 SQL JOIN처럼 특정 컬럼값을 기준으로 결합(how: inner/outer/left/right)
  • df1.join(df2)는 인덱스를 기준으로 결합 — 기준이 컬럼이냐 인덱스냐가 merge와 join의 핵심 차이
merge (컬럼 기준) vs join (인덱스 기준)
df1 = pd.DataFrame({'ID': [1, 2, 3], 'Name': ['Alice', 'Bob', 'Charlie']})
df2 = pd.DataFrame({'ID': [1, 2, 4], 'Age': [25, 30, 22]})

result = pd.merge(df1, df2, on='ID')                 # inner join (기본)
result_outer = pd.merge(df1, df2, on='ID', how='outer')

df3 = pd.DataFrame({'Name': ['Alice','Bob'], 'Age':[25,30]}, index=[1,2])
df4 = pd.DataFrame({'Salary': [50000, 60000]}, index=[1,2])
result_join = df3.join(df4)   # 인덱스를 기준으로 결합

  • histplot(분포), boxplot(사분위 분포), barplot(그룹 평균), scatterplot(상관관계) — hue 옵션으로 그룹별 색상 구분

다중 그래프(subplots)와 스타일 설정

  • fig, axes = plt.subplots(행, 열)로 한 화면에 여러 그래프를 격자로 배치, axes[i, j]에 각각 그래프를 그림
  • Seaborn 그래프도 ax=axes[i,j] 인자로 subplot 안에 배치 가능, plt.tight_layout()으로 간격 자동 조정
  • sns.set_style('whitegrid')·plt.style.use('ggplot')로 전체 그래프 스타일을 통일
subplots로 4개 그래프 한 화면에 배치
fig, axes = plt.subplots(2, 2, figsize=(10, 8))

corr = tips.select_dtypes(include=["float64", "int64"]).corr()
sns.heatmap(corr, annot=True, cmap="coolwarm", ax=axes[1, 0])
sns.barplot(x="day", y="total_bill", data=tips, ax=axes[1, 1])

plt.tight_layout()
plt.show()

# 스타일 통일
sns.set_style('whitegrid')     # 'darkgrid','white','dark','ticks' 중 선택
plt.style.use('ggplot')        # matplotlib 자체 스타일
